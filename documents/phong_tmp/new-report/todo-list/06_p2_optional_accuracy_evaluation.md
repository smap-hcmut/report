# P2 - Optional Accuracy Evaluation for Analytics

## Mục tiêu

Nếu còn thời gian, bổ sung một đánh giá định lượng cho tầng phân tích để tăng sức nặng học thuật ở phần thực nghiệm.

## Vì sao cần làm

Rubric cho phép “độ chính xác hoặc hiệu năng”. Nếu đã có benchmark hiệu năng tối thiểu, accuracy evaluation là phần cộng thêm rất mạnh cho bảo vệ và report.

## Hướng khả thi

1. Chọn một tập mẫu nhỏ đã gán nhãn thủ công.
2. Đánh giá một tác vụ cụ thể như sentiment hoặc intent.
3. Tính các chỉ số cơ bản: accuracy, precision, recall, F1 nếu làm được.
4. Ghi rõ quy mô dữ liệu và giới hạn của phép đo.

## Việc cần làm

1. Xác định bài toán nào thực sự khả thi để đánh giá.
2. Chuẩn bị sample set và nhãn tham chiếu.
3. Chạy pipeline hoặc module liên quan.
4. Tổng hợp kết quả thành bảng.

## Kết quả đầu ra mong muốn

- `analytics_accuracy_evaluation.md`
- bảng chỉ số và mô tả tập dữ liệu

## Ưu tiên

P2 - rất tốt nếu có, nhưng không nên chặn việc hoàn thiện report nếu thời gian gấp.
