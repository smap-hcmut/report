# Abstract

This thesis presents an evidence-based architectural and implementation study of the SMAP Social Media Analytics Platform, a multi-service system designed for social listening, analytics processing, knowledge retrieval, and real-time notification delivery. Instead of relying on high-level product descriptions alone, the study is grounded in source-code evidence collected from service manifests, configuration files, route definitions, migration scripts, test assets, and deployment descriptors available in the project workspace.

The thesis first identifies the actual technology stack of the system, including Go and Python service runtimes, PostgreSQL, Redis, Kafka, RabbitMQ, MinIO, Qdrant, OAuth2, JWT, Docker, and Kubernetes-related deployment artifacts. Based on this verified foundation, the system is analyzed in terms of functional requirements, non-functional requirements, use cases, business process flow, architectural boundaries, data design, API structure, and implementation of core features. Particular attention is paid to the separation between business context, ingest runtime, analytics processing, knowledge indexing, and notification delivery, as well as to the coexistence of multiple communication mechanisms in the current architecture.

The implementation study demonstrates how key capabilities such as OAuth2-based authentication, project lifecycle control, datasource and dry-run management, asynchronous analytics processing, semantic search and chat, and real-time notification delivery are realized in concrete files and functions. The evaluation chapter further shows that the system contains meaningful testing artifacts, contract checks, runtime decision-table tests, and architectural indicators of performance readiness, although fully controlled benchmark evidence is not yet available in the repository.

The main contribution of the thesis lies in reconstructing a consistent chain from requirements to design and from design to implementation using source-code evidence. By doing so, the thesis provides a reliable technical document for further engineering, maintenance, academic evaluation, and future extension of the SMAP platform.
